## Relatório de Entendimento — Xray V1

### Resumo do produto

Xray é um analisador de qualidade de especificações assistido por LLM. Recebe um texto livre (prompt, requisito ou briefing), avalia-o em 6 dimensões ponderadas, atribui um score 0–100 e devolve um diagnóstico estruturado (lacunas, ambiguidades, suposições, sugestões e uma versão melhorada sem inventar requisitos). O objetivo é fazer o utilizador especificar melhor *antes* de enviar a um LLM ou time de desenvolvimento.

### Fluxo principal (happy path)

1. Utilizador cola/digita a especificação e escolhe o tipo (prompt | requirement | briefing).
2. Frontend valida 10–10.000 chars e habilita o botão Analisar.
3. `POST /api/analyze` envia texto + tipo ao backend.
4. Backend monta system+user prompt, chama OpenRouter (temperature 0.2, timeout 25s).
5. LLM devolve JSON v1.0; backend valida com Pydantic (retry 1× se inválido) e recalcula `score.total`.
6. Frontend renderiza score badge, radar, dimensões, lacunas, ambiguidades, suposições, sugestões e before/after.
7. Análise é auto-guardada no LocalStorage; re-análise mostra delta de score.

### Riscos identificados

| Risco | Mitigação no código |
|---|---|
| LLM devolve JSON inválido | Strip de markdown + Pydantic + retry 1× com prompt de correção |
| Scores inconsistentes | temperature 0.2 + backend recalcula `score.total` |
| CORS bloqueia fetch | `XRAY_CORS_ORIGINS` configurável |
| Chart.js CDN fora | Pin de versão + fallback de mensagem |

### Decisões fixas (defaults D1–D5, sem perguntar)

Vanilla JS · Chart.js · rate limit 10 req/min/IP · UI PT-BR · suposições visíveis na V1.
